JWT

JSON Web Tokens is a pattern to exchange information securely between parties. JWT is split between three parts:

  • Header
  • Payload
  • Signature


Header

The header basically have information about the type of the token and the encryption used. On the example below we are setting the type of token to JWT of course, and the encryption algorithm will be HS256.


Payload

The Payload is where the information will be stored. Each These informations are called Claims and they are usually important data about the user that is authenticating or any additional information important for your project.

Registered claims

Registered claims are predefined claims that you can use, here are some examples:

  • iat : time the token was issued
  • exp : token expiration time
  • nbf : defines the time the token is valid (you can use to define a time in the future that the token starts to be valid)

On the example above, the token was issued at May,18th 2020 - 13:00:00 and it expires May, 18th, 2020 - 13:59:59. These would be the necessary parameters to define a token valid for one hour. The issuer would get the exp parameter and check if the token is still valid.

Signature

The Signature is where the JWT magic happens. By using the header, payload and a secret key, we are creating the third part of the JWT.

The signature is generated by the encryption algorithm chosen, in this example we are using HS256. The signature would be created using the following pattern:

HMACSHA256(
base64UrlEncode(header) + "." +
base64UrlEncode(payload),
"Xn2r5u8x/A?D(G+KbPeShVmYq3s6v9y$" )


The last part of the signature is the 256 bit key. This key will be private and only known by the issuer. This is the 'key' components that makes JWT secure. A signature will be created using all the information (header and payload) and the secret key. Since the secret key is included in the signature, the only way to change any information in the header or in the payload is by creating a new signature. If the hackers doesn't know the secret key, it's impossible to create a new and valid JWT.
